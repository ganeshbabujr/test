5. What is Parallel Hint and how it improve the performance?

Parallel hint instruct the optimizer to use multiple server processes to execute SQL statements. Prallel hint divide large task into smaller and processed multiple cpu threads to process at once instead of using single server process This can improve the performance while processing large volume of data to reduce the processing time and loading data or creating index 1.Splits large operations Full Table Scan Index Range Scan Table Joins Loading Data 2. Utilizing Multiple CPU Cores 3. Reducing processing time SELECT /*+ PARALLEL (table_alias [degree]) */ column1, column2 FROM table_name table_alias WHERE condition; INSERT /*+ PARALLEL (table_name [degree]) */ INTO table_name ... SELECT ... ; CREATE TABLE AS SELECT /*+ PARALLEL (table_alias [degree]) */ ... ; CREATE INDEX /*+ PARALLEL (table_name [degree]) */ index_name ON table_name (column_name); -- degree we use like 4,8,16 we can use maximum based on Orace CPU

INDEX Hints

we instruct otimiser to use specific index instead of using Full table scan. SELECT /*+ INDEX(table_alias index_name) */ columns FROM table_name table_alias WHERE conditions;

Append Hints

APPEND hint is used with the INSERT statement to enable direct-path insert, which allows Oracle to bypass the buffer cache and write data directly to datafiles. INSERT /*+ APPEND */ INTO target_table SELECT * FROM source_table;

No copy Hints

NOCOPY hint is used in PL/SQL procedures/functions to optimize the performance of IN OUT and OUT parameters. By default, Oracle uses pass-by-value for these parameters, it creates a temporary copy of the variable, and the copy is written back after the subprogram finishes. Using NOCOPY, Oracle use pass-by-reference, for large data like collections, records, or LOBs. PROCEDURE my_proc(p_data IN OUT NOCOPY my_table%ROWTYPE);

USE_HASH Hint

Forces Oracle to use a Hash Join between two tables SELECT /*+ USE_HASH(t2) */ ... FROM table1 t1 JOIN table2 t2 ON t1.id = t2.id; we use USE hash hints When joining large tables.

USE_NL Hint

Forces Oracle to use a Nested Loop Join between two tables SELECT /*+ USE_NL(t2) */ ... FROM table1 t1 JOIN table2 t2 ON t1.id = t2.id; When one table is small When indexes are available on the join column.

USE_MERG

USE_MERGE hint instructs Oracle to use a Merge Join to join two or more tables. SELECT /*+ USE_MERGE(table_alias) */ columns FROM table1 t1 JOIN table2 t2 ON t1.col = t2.col; When joining large datasets both tables having large data
